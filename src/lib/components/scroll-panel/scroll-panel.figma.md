## Overview

`ExtraScrollPanelComponent` — контейнер с кастомным кросс-браузерным скроллбаром. Оборачивает PrimeNG `p-scrollpanel` и заменяет нативную полосу прокрутки браузера на стилизованную в соответствии с темой. Используется, когда контент не помещается в отведённую область и нужна прокрутка по вертикали или горизонтали с единообразным внешним видом во всех браузерах.

Компонент соответствует Figma-компоненту `<ScrollPanel>` (nodeId `33:1560`, библиотека «UI Kit (DS) v2.0»). У компонента нет визуальных вариантов в Figma: его поведение определяется фиксированными размерами области и проецируемым содержимым.

## Props mapping

| Свойство | Тип | По умолчанию | Описание |
|----------|-----|--------------|---------|
| `height` | `string` | `'100px'` | Высота области прокрутки (любая CSS-единица); фиксированная высота включает вертикальный скроллбар, когда контент выше |
| `width` | `string` | `'100%'` | Ширина области прокрутки (любая CSS-единица); фиксированная ширина включает горизонтальный скроллбар, когда контент шире |
| `step` | `number` | `10` | Шаг прокрутки при нажатии клавиш со стрелками (в пикселях) |

## Variants

Визуальных вариантов в Figma нет. Направление прокрутки определяется соотношением размеров области (`height`/`width`) и габаритов проецируемого содержимого.

### Вертикальная прокрутка

Фиксированная высота меньше высоты контента — появляется вертикальный скроллбар.

```html
<extra-scroll-panel height="100px">
  <p>Длинный текст или список, который выше отведённой области и прокручивается по вертикали.</p>
</extra-scroll-panel>
```

### Горизонтальная прокрутка

Контент без переносов шире области — появляется горизонтальный скроллбар.

```html
<extra-scroll-panel height="80px">
  <p style="white-space: nowrap; margin: 0">
    Длинная строка без переносов, которая выходит за ширину контейнера и прокручивается по горизонтали.
  </p>
</extra-scroll-panel>
```

### Прокрутка в обоих направлениях

Содержимое шире и выше области — появляются оба скроллбара.

```html
<extra-scroll-panel height="200px" width="50%">
  <div style="width: max-content">
    <p style="white-space: nowrap; margin: 0 0 0.5rem 0">Строка таблицы шире контейнера…</p>
    <p style="white-space: nowrap; margin: 0 0 0.5rem 0">…и таких строк больше, чем помещается по высоте.</p>
  </div>
</extra-scroll-panel>
```

### Кастомный шаг прокрутки

```html
<extra-scroll-panel height="120px" [step]="30">
  <p>Прокрутка стрелками клавиатуры выполняется шагом 30px вместо стандартных 10px.</p>
</extra-scroll-panel>
```

## Slots

| Слот | Синтаксис | Описание |
|------|-----------|---------|
| Содержимое | `<ng-content>` | Произвольное прокручиваемое содержимое — размещается напрямую между тегами `<extra-scroll-panel>` |

Именованных слотов и структурных директив компонент не имеет. Любая разметка между открывающим и закрывающим тегами попадает в безымянный `<ng-content>` и становится прокручиваемой областью.

## Do / Don't

**Do:**
- Задавайте фиксированную `height` (и при необходимости `width`), чтобы область могла прокручиваться — без ограничения размера скроллбар не появится.
- Используйте для длинных списков, текстов и таблиц внутри карточек, панелей и сайдбаров, где нужен единый вид скроллбара во всех браузерах.
- Для горизонтальной прокрутки задавайте содержимому `white-space: nowrap` или фиксированную ширину контента.

**Don't:**
- Не используйте компонент только ради факта прокрутки — для нативного поведения без кастомного вида достаточно CSS `overflow`.
- Не путайте со скроллом внутри `Dialog`/`Drawer` — это отдельные поверхности; `extra-scroll-panel` лишь делает вложенную область прокручиваемой.
- Не задавайте `height: 100%` без ограниченного по высоте родителя — область растянется и прокрутка не возникнет.
